## Codebase Patterns
- Pipeline step definitions live in `packages/core/src/types/pipeline.ts` (SOP_STEPS, SOP_STEP_NAMES)
- UI step names are duplicated in pipeline-step-nav.tsx (STEP_NAMES), pipeline-progress.tsx (SHORT_STEP_NAMES), and pipeline-view.tsx (STEP_TITLES) — all must stay in sync
- Static API routes for specific steps live at `apps/web/src/app/api/projects/[id]/stages/<N>/<action>/route.ts`
- The `[step]/run/route.ts` dynamic route handles all step run requests; export step is blocked there
- Worker export-html handler references step numbers — keep in sync with UI
- Step completion flow: pipeline-view.tsx calls API routes which update DB stages and project currentStage
- Optional steps (currently step 11 = Edit for Style) use `/stages/<N>/complete` endpoint
- Skippable steps (currently step 9 = Integrate Critiques) use `/stages/<N>/skip` endpoint
- Critiques flow: step 8 (Devil's Advocate) → critiques/select API → either step 9 (Integrate) or skip to step 12 (Export)
